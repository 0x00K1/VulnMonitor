VulnMonitor User Guide

Your comprehensive guide to mastering VulnMonitor

Introduction

Welcome to **VulnMonitor**, your go-to application for monitoring and managing software vulnerabilities. This guide will help you get started and make the most out of VulnMonitor's features.

Installation

VulnMonitor is distributed as an executable for Windows and a JAR file for Linux and macOS. Follow the steps below to install and set up the application:

  1. Download the Application: Visit our [GitHub Releases](https://github.com/0x00K1/VulnMonitor/releases) page to download the appropriate file for your operating system:
    • Windows: Download the `VulnMonitor.exe` file.
    • Linux/macOS: Download the `VulnMonitor.jar` file.
  2. Run the Application:
    • Windows: Double-click the downloaded `VulnMonitor.exe` file to launch the application.
    • Linux/macOS: Open your terminal, navigate to the directory where the `VulnMonitor.jar` file was downloaded, and execute the following command: 
      java -jar VulnMonitor.jar

For the latest updates and detailed instructions, visit the VulnMonitor GitHub Repository.

Getting Started

After installation, familiarize yourself with the main dashboard and navigation menu. Here are some initial steps to get you started:

  • Login: Use your registered credentials to log into the application.
  • Explore the Dashboard: The dashboard provides an overview of the latest vulnerabilities and system status.
  • Set Up Your preferences: Navigate to the Settings section to customize your user configuration, including notification, add/remove OS, product and interface themes.
  • Configure Filters: Use the Filters feature to tailor the CVE data displayed based on your specific criteria.
  • Create Alerts: Set up personalized alerts to receive notifications about new or critical vulnerabilities.

Key Features

VulnMonitor offers a range of powerful features to help you stay on top of vulnerabilities:

  • Real-Time Monitoring: Continuously fetches the latest CVE data from trusted sources to keep you informed.
  • Customizable Alerts: Set up alerts based on specific criteria such as severity level, affected products, or CVE identifiers.
  • Comprehensive Data Display: View detailed information about each vulnerability, including descriptions, severity scores, and references.
  • Efficient Data Management: Securely stores and manages large volumes of CVE data, ensuring quick access and retrieval.
  • User-Friendly Interface: Intuitive design developed with Java Swing, making it easy for users of all technical backgrounds to navigate and utilize.
  • Cross-Platform Compatibility: Available on Windows, macOS, and Linux, ensuring broad accessibility.
  • Dark Mode: Customize the interface with dark mode for a comfortable viewing experience.

Using Filters

Filters help you narrow down the vulnerabilities to those that matter most to you. Here's how to use them:

  1. Access the Filters Menu: Click on the "Filters" link in the main navigation menu to open the Filters interface.
  2. Select Criteria: Choose the desired criteria for filtering CVEs, such as:
    • Operating System (e.g., Windows, Linux, macOS)
    • Severity Level (e.g., Low, Medium, High, Critical)
    • Affected Products
    • Include Resolved CVEs
    • Include Rejected CVEs
  3. Apply Filters: After selecting your criteria, click the "Apply" button to filter the CVE data displayed on the dashboard.
  4. Save Filter Sets: For frequently used filters, you can save filter sets for quick access in the future.
  5. Clear Filters: To remove all applied filters, click the "Clear" or "Reset" button.

Viewing CVEs

To view detailed information about a specific CVE:

  1. Navigate to the CVE Table: Click on the "CVE Table" link in the main navigation menu to view the list of current vulnerabilities.
  2. Locate the CVE: Use the search bar or scroll through the table to find the CVE you are interested in.
  3. Select the CVE: Click on the CVE ID or the "View Details" button associated with the desired CVE.
  4. Review Details: A detailed view will appear, displaying comprehensive information about the CVE, including:
    • Description
    • Severity Score (CVSS)
    • Affected Products
    • Platforms
    • Published Date
    • References and External Links
  5. Take Action: From the detailed view, you can choose to archive the CVE, set up alerts, or export the information for further analysis.

Alerts and Notifications

Stay informed with VulnMonitor's alert system:

  1. Access Alerts: Click on the "Alerts" link in the main navigation menu to open the Alerts interface.
  2. Create a New Alert: Click the "Create Alert" button to define a new alert.
  3. Define Alert Criteria: Specify the conditions under which you want to receive notifications, such as:
    • Specific CVE IDs
    • Severity Levels
    • Affected Products
    • Operating Systems
  4. Select Notification Methods: Choose how you want to receive alerts:
    • System Notifications (in-app popups)
    • Email Notifications
    • Audio Alerts
  5. Save the Alert: After configuring your alert, click the "Save" button to activate it.
  6. Manage Existing Alerts: In the Alerts interface, you can view, edit, or delete your existing alerts as needed.
  7. Receive Notifications: When a CVE matching your alert criteria is detected, you will receive notifications based on your selected methods.

Managing User Settings

Customize your VulnMonitor experience by managing your user settings:

  1. Access Settings: Click on the "Settings" link in the main navigation menu to open the Settings interface.
  2. General Settings: Configure application-wide preferences, including:
    • Launch on Startup: Enable or disable the application launching automatically when your system starts.
    • Dark Mode: Toggle between light and dark themes for the interface.
  3. Fetcher Settings: Adjust how VulnMonitor fetches CVE data:
    • Auto-Fetch Interval: Set the frequency (in minutes) at which the application automatically fetches new CVE data.
    • Start/Stop Fetching: Manually initiate or halt the automatic data fetching process.
  4. Filter Settings: Manage your saved filter criteria for CVE data display.
  5. Alerts Settings: Customize how you receive alerts, including enabling/disabling sound alerts and system notifications.
  6. Archive Settings: Set limits on the number of CVEs that can be archived and manage archived data.
  7. Notifications Settings: Refresh, delete, or manage individual notifications.
  8. Account Settings: Update your account information, such as username, email address, and password.
  9. Save Changes: After making your desired changes, click the "Apply" button to save them.
  10. Discard Changes: If you decide not to keep the changes, click the "Cancel" button to revert.

Troubleshooting

If you encounter any issues, try the following troubleshooting steps:

  • Application Not Launching:
    • Ensure that Java Runtime Environment (JRE) is installed and up to date.
    • Verify that your system meets the minimum hardware requirements.
    • Restart your computer and try launching the application again.
  • Unable to Fetch CVE Data:
    • Check your internet connection to ensure it is stable.
    • Verify that the NVD and MITRE APIs are accessible by visiting their websites.
    • Increase the fetch interval in Settings to reduce the load on the APIs.
  • Login Issues:
    • Ensure that you are entering the correct username and password.
    • If you have forgotten your password, use the "Forgot Password" feature to reset it.
    • Check if your account has been locked due to multiple failed login attempts and contact support if necessary.
  • Notifications Not Working:
    • Ensure that notifications are enabled in the Alerts settings.
    • Check your email settings if you are not receiving email alerts.
    • Verify that your system's notification settings allow alerts from VulnMonitor.
  • Performance Issues:
    • Close unnecessary applications running in the background to free up system resources.
    • Adjust the fetch interval to reduce the frequency of data retrieval.
    • Update to the latest version of VulnMonitor to benefit from performance optimizations.
  • Application May Not Run Due To:
    • No Network Connection: Ensure your device is connected to the internet.
    • Incorrect Date/Time: Verify that your system's date and time settings are correct.
    • Server Host Problems: This includes issues with the Database or CVE Fetcher. Ensure that the servers are running and accessible.
  • Contact Support: If the issue persists after attempting the above steps, refer to the "Contact Support" section for further assistance.
